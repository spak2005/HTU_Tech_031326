---
name: scope-changes-incrementally
description: User wants changes scoped to one page/area at a time, not spread across the whole codebase at once
type: feedback
---

Don't make sweeping changes across the entire codebase at once. Work on one page/area at a time, get it working well, then expand to other pages.

**Why:** User prefers incremental, focused work rather than broad changes that are hard to review and test.

**How to apply:** When given a task that could touch many files, ask which area to start with or scope the first pass to the most relevant page/component.
